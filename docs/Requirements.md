## 1. Введение

### 1.1 Цель
Цель проекта — разработка веб-приложения **CollabRoom**, которое позволит пользователям создавать и использовать онлайн-комнаты для видеосвязи, аудиообщения и текстовых чатов.  
Основное предназначение системы — облегчить процесс удалённого взаимодействия для учебных групп, небольших рабочих команд и любых сообществ, которым требуется быстрая и удобная форма совместной коммуникации.  

### 1.2 Область применения
Приложение будет использоваться в следующих сценариях:
- онлайн-занятия и консультации для студентов и преподавателей;
- митинги и планёрки для малых команд;
- совместное обсуждение проектов или идей;
- дружеские встречи в формате видеочата.  

CollabRoom не предназначен для крупных конференций и трансляций с тысячами участников. Основной акцент сделан на **простоте, доступности и минимальном времени подключения**.  

### 1.3 Определения, сокращения и аббревиатуры
- **MVP** — минимально жизнеспособный продукт.  
- **WebRTC** — набор API для организации потоковой передачи аудио и видео в реальном времени между браузерами. 
- **REST API** — архитектурный стиль взаимодействия клиент-сервер.  
- **WebSocket** — протокол для двунаправленной связи в реальном времени.  

### 1.4 Ссылки
- [WebRTC API (MDN)](https://developer.mozilla.org/en-US/docs/Web/API/WebRTC_API)  
- [Spring Boot Documentation](https://spring.io/projects/spring-boot)  
- [PostgreSQL Documentation](https://www.postgresql.org/docs/)  
- [WebSocket API (MDN)](https://developer.mozilla.org/en-US/docs/Web/API/WebSockets_API)  

### 1.5 Обзор
Backend: **Java (Spring Boot) + PostgreSQL**.  
Frontend: **React/TypeScript или Vue/TypeScript**.  
Основные коммуникационные технологии: **WebRTC для медиа, WebSocket для текстовых чатов и синхронизации событий**.

## 2. Общее описание

### 2.1 Перспектива продукта
CollabRoom будет представлять собой браузерное приложение с простым интерфейсом. Пользователь сможет:
- зарегистрироваться и войти в систему;
- создать комнату и пригласить участников;
- подключиться по ссылке к существующей комнате.  

Продукт не требует установки дополнительного ПО и работает полностью в браузере.  

#### 2.1.1 Программные интерфейсы
- **WebRTC API** — организация потоковой передачи видео/аудио.  
- **Spring Boot REST API** — обработка запросов, управление комнатами и пользователями.  
- **PostgreSQL** — хранение данных о пользователях, комнатах и чатах.  
- **WebSocket API** — передача сообщений и событий в реальном времени.  

#### 2.1.2 Пользовательские интерфейсы
Интерфейс включает:
- форму регистрации и входа;  
- главную страницу (создание комнаты, вход по ссылке);  
- экран комнаты (видеопоток, чат, список участников, включение/выключение камеры).  

#### 2.1.3 Интерфейсы связи
- **P2P соединения через WebRTC**;  
- **WebSocket соединение** для чата и синхронизации событий;

#### 2.1.4 Ограничения памяти
- Поддержка до **10 участников** в комнате одновременно.  
- Нет сохранения медиафайлов (видеозаписей).  
- Минимальное хранение сообщений (только текст чата).  

### 2.2 Функции продукта
Основные функции:
- регистрация и аутентификация;  
- создание комнаты;  
- подключение к комнате по ссылке;  
- видеосвязь и аудиосвязь;  
- чат;  
- управление устройствами (микрофон/камера/захват экрана);  
- удаление участников администратором.  

### 2.3 Характеристики пользователей
- **Обычные пользователи**: студенты, преподаватели, команды (низкий уровень технической подготовки, нужен простой интерфейс).  
- **Администраторы**: более опытные пользователи, которые управляют комнатами и их участниками.  

### 2.4 Ограничения
- Поддержка только современных браузеров (Chrome, Firefox, Edge, Safari). 
- Нет интеграции с внешними сервисами.  

### 2.5 Предположения и зависимости
- Все пользователи имеют устройства с камерой и микрофоном.  
- Браузеры поддерживают WebRTC.  

### 2.6 Разделение требований
Будущие версии могут включать:
- запись встреч;  
- интеграцию с календарями, интерактивной доской и блокнотом;  
- масштабирование на сотни участников.

## 3. Специфические требования

### 3.1 Внешние интерфейсы
- Веб-браузер.  
- REST API и WebSocket.  
- База данных PostgreSQL.  

### 3.2 Функции
1. Регистрация и вход.  
2. Создание комнаты.  
3. Подключение участников по ссылке.  
4. Видео/аудио общение.  
5. Чат.  
6. Управление микрофоном/камерой.  
7. Отображение участников.  
8. Удаление участников.  

### 3.3 Требования к производительности
- Поддержка до 10 участников.  
- Минимальная задержка (<500 мс).  
- Чат — мгновенная доставка сообщений.  

### 3.4 Логические требования к данным
- **Пользователи**: id, email, пароль (BCrypt), роль.  
- **Комнаты**: id, владелец, список участников, ссылка.  
- **Сообщения**: id, id комнаты, автор, текст, timestamp.  

### 3.5 Атрибуты системы

#### 3.5.1 Надёжность
Система должна сохранять соединение при кратковременных перебоях в сети.  

#### 3.5.2 Доступность
Продукт должен быть доступен 24/7 при работающих серверах.  

#### 3.5.3 Безопасность
- Пароли в хэшированном виде (BCrypt).  
- Доступ по уникальным ссылкам.  

#### 3.5.4 Поддерживаемость
Архитектура разделена на сервисы. Обновления могут выполняться по отдельным сервисам.  

#### 3.5.5 Переносимость
Работа на разных ОС (Windows, Linux, macOS) через браузер.  

#### 3.5.6 Другие характеристики
Простой интерфейс: подключение к комнате за 2–3 клика.
